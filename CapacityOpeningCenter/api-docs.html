快速开始

步骤1：获取API密钥

在使用API前，您需要先注册并获取API密钥。请按照以下步骤操作：

  1. 注册并登录医疗AI开放平台账号
  2. 在控制台中创建应用
  3. 查看并复制您的API密钥

API_KEY = "your_api_key_here"

步骤2：发送API请求

使用您的API密钥发送请求到我们的API端点。以下是一个简单的示例：

POST https://api.medical-ai.com/v1/text/analyze

请求头

Content-Type: application/json
Authorization: Bearer {your_api_key}

请求体

{  
  "text": "患者男，65岁，高血压病史10年，今日测量血压150/95mmHg",
  "task_type": "medical_summary"
}

响应

{  
  "status": "success",
  "data": {
    "summary": "老年男性患者，有长期高血压病史，血压控制不佳",
    "entities": [
      {"type": "disease", "value": "高血压", "confidence": 0.98},
      {"type": "age", "value": "65岁", "confidence": 1.0}
    ]
  },
  "request_id": "req_123456789"
}

步骤3：集成到您的应用

使用我们提供的SDK或直接调用API，将医疗AI能力集成到您的应用中。

JavaScript示例 

const fetch = require('node-fetch');

async function analyzeMedicalText(text) {
  const response = await fetch('https://api.medical-ai.com/v1/text/analyze', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer ' + process.env.API_KEY
    },
    body: JSON.stringify({
      text: text,
      task_type: 'medical_summary'
    })
  });
  
  return await response.json();
}

// 使用示例
analyzeMedicalText('患者男，65岁，高血压病史10年').then(data => {
  console.log(data);
});

Python示例 

import requests
import os

def analyze_medical_text(text):
    url = "https://api.medical-ai.com/v1/text/analyze"
    headers = {
        "Content-Type": "application/json",
        "Authorization": f"Bearer {os.environ.get('API_KEY')}"
    }
    data = {
        "text": text,
        "task_type": "medical_summary"
    }
    response = requests.post(url, headers=headers, json=data)
    return response.json()

# 使用示例
result = analyze_medical_text("患者男，65岁，高血压病史10年")
print(result)

认证与授权

API密钥认证

我们的API使用Bearer Token认证方式，您需要在请求头中包含有效的API密钥。

Authorization: Bearer your_api_key_here

安全提示

请勿在客户端代码中暴露API密钥，建议在服务端进行API调用。如果API密钥不慎泄露，请立即在控制台中撤销并重新生成。

OAuth 2.0 认证

对于需要用户授权的应用，我们支持OAuth 2.0认证流程。

授权码流程

  1. 引导用户到授权页面：https://auth.medical-ai.com/oauth/authorize
  2. 用户授权后，重定向到您的回调URL并附带授权码
  3. 使用授权码换取访问令牌和刷新令牌
  4. 使用访问令牌调用API
# 步骤3：换取访问令牌
POST https://auth.medical-ai.com/oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&
code=your_authorization_code&
client_id=your_client_id&
client_secret=your_client_secret&
redirect_uri=your_redirect_uri

医学文本分析API

POST https://api.medical-ai.com/v1/text/analyze

对医学文本进行智能分析，包括结构化、实体识别、关系提取等功能。

请求参数

参数名 类型 必填 描述 示例
text string 是 要分析的医学文本 患者男，65岁，高血压病史10年
task_type string 是 任务类型：medical_summary（病历摘要）、entity_recognition（实体识别）、relation_extraction（关系提取）、deidentify（去隐私化） medical_summary
options object 否 任务特定选项 {"include_confidence": true}

响应格式

{  
  "status": "success",
  "data": {
    "summary": "老年男性患者，有长期高血压病史",
    "entities": [
      {"type": "disease", "value": "高血压", "confidence": 0.98},
      {"type": "age", "value": "65岁", "confidence": 1.0}
    ],
    "relations": [
      {"subject": "患者", "predicate": "有", "object": "高血压病史"}
    ]
  },
  "request_id": "req_123456789",
  "timestamp": "2023-06-15T10:30:00Z"
}
POST https://api.medical-ai.com/v1/text/standardize

将非标准医学术语转换为标准术语，支持ICD-10、SNOMED CT、UMLS等标准。

请求参数

参数名 类型 必填 描述 示例
terms array 是 要标准化的术语列表 ["高血压", "心梗"]
standard string 是 目标标准：ICD-10、SNOMED_CT、UMLS、CHV ICD-10

响应格式

{  
  "status": "success",
  "data": {
    "standardized_terms": [
      {
        "original": "高血压",
        "standardized": "I10",
        "standardized_name": "原发性高血压",
        "confidence": 0.98
      },
      {
        "original": "心梗",
        "standardized": "I21",
        "standardized_name": "急性心肌梗死",
        "confidence": 0.95
      }
    ]
  },
  "request_id": "req_123456789"
}

速率限制

为确保服务的稳定性和公平使用，我们对API调用实施速率限制。不同等级的账户有不同的限制策略。

账户等级 请求频率（每分钟） 每日调用量
免费账户 60 5,000
标准账户 300 100,000
企业账户 1,000+ 1,000,000+

响应头中的速率限制信息

API响应头中包含以下速率限制相关字段：

  • X-RateLimit-Limit: 当前时段允许的最大请求数
  • X-RateLimit-Remaining: 当前时段剩余的可用请求数
  • X-RateLimit-Reset: 速率限制重置的时间戳（秒）

错误码

当API调用失败时，我们会返回相应的错误码和错误信息。以下是常见的错误码列表：

状态码 错误码 描述 解决方案
400 BAD_REQUEST 请求参数错误或格式不正确 检查请求参数是否符合API文档要求
401 UNAUTHORIZED 认证失败，API密钥无效或过期 检查API密钥是否正确，必要时重新生成
403 FORBIDDEN 无权访问请求的资源或功能 确认您的账户权限是否足够
404 NOT_FOUND 请求的资源不存在 检查API端点是否正确
429 RATE_LIMIT_EXCEEDED 超出速率限制 稍后重试或升级账户以提高限制
500 SERVER_ERROR 服务器内部错误 请稍后重试，如问题持续请联系技术支持
