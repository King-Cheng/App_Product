快速开始

1

注册与登录

要开始使用医疗AI开放平台，首先需要注册一个账号：

  1. 访问平台首页，点击右上角的"注册"按钮
  2. 填写必要的注册信息，包括用户名、邮箱和密码
  3. 完成邮箱验证
  4. 进行实名认证（根据平台规定，某些功能需要实名认证）

注册成功后，使用您的账号和密码登录平台。

提示

为了您的账号安全，建议设置强密码并定期更换。

2

创建应用

登录成功后，您需要创建一个应用来管理API调用：

  1. 登录控制台
  2. 在左侧导航栏选择"应用管理"
  3. 点击"创建应用"按钮
  4. 填写应用名称、描述和选择应用类型
  5. 点击"确定"完成创建

创建应用界面示例

3

获取API密钥

应用创建成功后，您需要获取API密钥以调用平台API：

  1. 在"应用管理"页面找到您创建的应用
  2. 点击"查看详情"或进入应用设置
  3. 找到"API密钥"部分，点击"生成密钥"
  4. 系统将生成API Key和Secret Key，请妥善保管

重要提示

API密钥是您访问平台API的凭证，请妥善保管，不要分享给他人。如发现密钥泄露，请立即在控制台进行重置。

4

开始调用API

获取API密钥后，您可以开始调用平台API：

Python示例代码：

import requests
import json

# API密钥
api_key = "your_api_key"
api_secret = "your_api_secret"

# API地址
url = "https://api.medicalaicloud.com/v1/text-analysis"

# 请求头
headers = {
    "Content-Type": "application/json",
    "Authorization": f"Bearer {api_key}"
}

# 请求数据
data = {
    "text": "患者，男，65岁，有高血压病史10年，今日测量血压为150/95mmHg。",
    "analysis_type": "medical_terms"
}

# 发送请求
response = requests.post(url, headers=headers, data=json.dumps(data))

# 处理响应
if response.status_code == 200:
    result = response.json()
    print("分析结果：", json.dumps(result, indent=2, ensure_ascii=False))
else:
    print(f"请求失败：{response.status_code}")
    print("错误信息：", response.json())

JavaScript示例代码：

// API密钥
const apiKey = "your_api_key";
const apiSecret = "your_api_secret";

// API地址
const url = "https://api.medicalaicloud.com/v1/text-analysis";

// 请求头
const headers = {
    "Content-Type": "application/json",
    "Authorization": `Bearer ${apiKey}`
};

// 请求数据
const data = {
    text: "患者，男，65岁，有高血压病史10年，今日测量血压为150/95mmHg。",
    analysis_type: "medical_terms"
};

// 发送请求
fetch(url, {
    method: "POST",
    headers: headers,
    body: JSON.stringify(data)
})
.then(response => {
    if (!response.ok) {
        throw new Error(`请求失败: ${response.status}`);
    }
    return response.json();
})
.then(result => {
    console.log("分析结果:", result);
})
.catch(error => {
    console.error("错误:", error);
});

API调用基础

API请求格式

所有API请求均使用HTTP/HTTPS协议，主要支持以下方法：

  • GET - 用于获取资源信息
  • POST - 用于创建新资源或提交数据处理请求
  • PUT - 用于更新资源
  • DELETE - 用于删除资源

大多数API请求的请求体和响应体都使用JSON格式。

API认证方式

平台支持以下认证方式：

1. API密钥认证

使用在控制台获取的API密钥进行认证，在请求头中添加：

Authorization: Bearer YOUR_API_KEY

2. OAuth 2.0认证

对于企业用户，平台还支持OAuth 2.0认证，提供更高级的安全保障和权限控制。

API响应格式

所有API响应都包含以下基本字段：

{
    "code": 0,                // 状态码，0表示成功
    "message": "success",    // 状态消息
    "data": {                // 响应数据，根据API不同而不同
        // ...
    },
    "request_id": "uuid"      // 请求唯一标识符，用于问题排查
}

如果请求失败，code将是非零值，message会包含错误描述。

最佳实践

性能优化

使用批量请求

尽可能使用批量API处理多条数据，减少请求次数，提高效率。

实现本地缓存

对于频繁使用且不常变化的数据，实现本地缓存，避免重复调用API。

异步处理

对于耗时较长的操作，使用异步处理机制，避免阻塞主流程。

安全最佳实践

保护API密钥

不要在代码中硬编码API密钥，使用环境变量或专门的密钥管理服务存储。

使用HTTPS

始终使用HTTPS协议进行API调用，确保数据传输安全。

定期轮换密钥

定期更换API密钥，特别是在团队成员变动时，以降低泄露风险。

常见问题

如何查看API使用情况？

登录控制台后，在"应用管理"页面选择对应的应用，可以查看API调用次数、成功率、响应时间等使用情况统计。

API调用失败了怎么办？

首先检查返回的错误码和错误信息，对照API文档中的错误码说明。常见的失败原因包括：API密钥无效、请求参数错误、调用频率超限等。如果问题持续，请联系技术支持。

如何申请提高API调用额度？

企业用户可以在控制台提交工单申请提高API调用额度，需要说明提高额度的原因和预期使用场景。我们的技术团队会在1-3个工作日内审核并回复。

如何获取技术支持？

平台提供多种技术支持方式：在线客服（工作日9:00-18:00）、技术支持邮箱（support@medicalaicloud.com）、企业用户专属技术顾问等。

如果您在使用过程中遇到其他问题，请参考常见问题页面或联系我们的技术支持团队。
